1) 데이터 베이스 설계 시 고려사항
- 무결성
- 일관성
- 회복 
- 보안 
- 효율성
- 데이터 베이스 확장
2) 데이터베이스 설계 순서

| 순서       | 설명                                       |
| :------- | ---------------------------------------- |
| 요구 조건 분석 | 요구 조건 명세서 작성                             |
| 개념적 설계   | 독립적인 개볌 스키마 모델링, 토랜잭션 모델링                |
| 논리적 설게   | 목표 DBMS에 맞는 논리 스키마 설계, 논리적 데이터베이스 구조로 매핑 |
| 물리적 설계   | 목표DBMS에맞는 몰리적 구조의 데이터로 변환                |
| 구현       | 목표 DBMS의 DDL로 데이터베이스생성, 트랜잭션 작성          |
